1. 开启 Windows 功能（必须）

  1. 打开 控制面板

  2. 进入：程序和功能

  3. 点击左侧：打开或关闭 Windows 功能

  4. 勾选以下两项：

   - ✅ 适用于 Linux 的 Windows 子系统

   - ✅ 虚拟机平台（可能显示英文 Virtual Machine Platform）

   - ✅ Windows虚拟机监控程序平台

  1. 点击 确定

  2. 按提示 重启电脑

2. 确认虚拟化已启用（推荐检查）

重启后确认 CPU 虚拟化功能已开启：

  1. 桌面右键任务栏 → 进入 任务管理器

  2. 打开：性能 → CPU

  3. 查看右侧信息：

   - 若看到 “虚拟化：已启用” ✅ 说明满足 WSL2 运行条件

若显示“未启用”，通常需要进 BIOS/UEFI 打开 Intel VT-x / AMD-V（SVM）等虚拟化选项。

6.移动 Linux（WSL）安装位置（推荐）

默认情况下，WSL 发行版的数据通常在 C 盘，长期使用后可能占用 30GB 甚至更多空间。

如果你的 C 盘空间不宽裕，建议把 WSL 发行版迁移到其他盘（如 D/E 盘）。

思路：先导出（export）发行版到 tar 包 → 注销（unregister）旧发行版 → 再导入（import）到新路径并指定 WSL2。
